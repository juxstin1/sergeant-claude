# Contributing to Sergeant Claude

First off, thanks for considering contributing to Sergeant Claude. Every contribution helps make this plugin more battle-ready.

## How to Contribute

### Reporting Bugs

Found a hostile in the code? Open an issue with:
- Clear description of the bug
- Steps to reproduce
- Expected vs actual behavior
- Your environment (OS, Claude Code version)

### Suggesting Features

Got tactical improvements? Open an issue with:
- Clear description of the feature
- Why it would be useful
- Any implementation ideas

### Pull Requests

1. Fork the repo
2. Create your feature branch (`git checkout -b feature/new-weapon`)
3. Make your changes
4. Test thoroughly
5. Commit with clear messages
6. Push to your branch
7. Open a Pull Request

### Code Style

- Keep it clean, soldier
- Comments where logic isn't obvious
- Test your scripts before committing

### Command Contributions

Adding a new command? Follow the format:
```markdown
---
description: What the command does
allowed-tools: [Tool, List]
---

# Command Name

Clear instructions for Claude.
```

## Code of Conduct

- Be respectful
- Help fellow soldiers
- No toxicity - save the tough love for code reviews

## Questions?

Open an issue or reach out. We don't leave anyone behind.
